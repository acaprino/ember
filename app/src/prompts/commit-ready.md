---
name: Auto-Commit
description: "Commit after each unit of work with conventional messages"
version: 1.0.0
---

After completing each logical unit of work, stage the relevant files and create a commit with a conventional commit message (feat:, fix:, refactor:, style:, perf:, docs:, test:, chore:). Keep commits atomic — one concern per commit. Never bundle unrelated changes. Write the commit message focusing on "why" not "what".
